# JobAgent V2 V1 Reference Map

This file is a reference map, not a migration checklist. V1 code is read-only
reference material. The default action is clean V2 reimplementation informed by
V1 lessons. No V1 code may be copied merely because it appears in this document.

When implementing a V2 feature:

1. Design the V2 feature from the authoritative planning documents.
2. Search this V1 reference map for related logic.
3. Inspect the listed V1 files and tests for useful ideas, edge cases,
   invariants, and failure modes.
4. Reimplement the V2 feature cleanly around the V2 architecture.
5. Copy source code only when the component is a tiny, isolated, deterministic
   primitive and copying it is clearly safer than rewriting it.

V2 usage values:

- `reference only`: inspect for tests, schemas, lessons, or edge cases.
- `adapt concept`: reimplement the idea around V2 architecture.
- `copy isolated primitive`: copying may be acceptable after reviewing tests and
  dependency assumptions.
- `ignore`: V1-only infrastructure or product ballast.
